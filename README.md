# Tourfecto Backend

باك-إند حقيقي (Node.js + Express + قاعدة بيانات SQLite) لمنصة Tourfecto، فيه:
- تسجيل حسابات ودخول حقيقي (عميل / مورد) بكلمات مرور مشفّرة وتوكنات JWT
- إدارة الأماكن (فنادق/مطاعم/معالم) — بحث، فلترة، إضافة/تعديل/حذف للمورد
- نظام تقييمات حقيقي مرتبط بحساب المستخدم ومتخزن بشكل دائم
- لوحة إحصائيات للمورد (مشاهدات، عدد التقييمات، متوسط التقييم) من بيانات حقيقية

## 1. التشغيل محليًا

```bash
cd tourfecto-backend
npm install
cp .env.example .env
# افتح .env وغيّر JWT_SECRET لقيمة عشوائية طويلة
npm run seed     # يعبّئ قاعدة البيانات ببيانات تجريبية (اختياري لكن مستحسن)
npm start
```

السيرفر هيشتغل على `http://localhost:4000` (أو البورت اللي حددته في `.env`).

بعد الـ seed، هيبقى عندك حسابين تجريبيين:
- **مورد:** `vendor@tourfecto.demo` / `vendor123`
- **عميل:** `customer@tourfecto.demo` / `customer123`

## 2. رفعه على استضافة حقيقية

يشتغل مباشرة على أي منصة بتدعم Node.js، زي:
- **Render** أو **Railway**: اربط الـ repo، وحدد `npm start` كأمر التشغيل، وضيف متغيرات البيئة من `.env.example` في إعدادات المشروع
- تأكد إن فولدر `data/` (اللي فيه ملف قاعدة البيانات) دايم موجود ومحفوظ (persistent disk)، وإلا هتفقد البيانات عند كل إعادة نشر

> ملاحظة: SQLite ممتاز للبداية وحتى آلاف الزيارات، لكن لو التطبيق كبر ومحتاج أكتر من سيرفر شغال في نفس الوقت، الخطوة الطبيعية بعد كده هي الانتقال لـ PostgreSQL (بنية الجداول في `src/schema.sql` قريبة جدًا من SQL القياسي وتقدر تحولها بسهولة).

## 3. ربطه بموقع Tourfecto (الفرونت إند)

الفرونت إند اللي عندك دلوقتي (ملف HTML واحد) بيخزن البيانات في المتصفح بس. لازم تعدّله عشان يكلم الـ API ده بدل البيانات الثابتة، عن طريق `fetch()` لكل الروابط تحت. لو حابب أعمل الربط ده كخطوة تالية، قول لي.

## 4. توثيق الـ API

جميع الطلبات (عدا `/register` و`/login`) اللي تتطلب تسجيل دخول لازم تبعت الهيدر:
```
Authorization: Bearer <token>
```

### تسجيل حساب
```
POST /api/auth/register
Body: { "name", "email", "password", "role": "customer" | "vendor", "businessName"? }
Response: { token, user }
```

### تسجيل الدخول
```
POST /api/auth/login
Body: { "email", "password" }
Response: { token, user }
```

### بيانات المستخدم الحالي
```
GET /api/auth/me     (يتطلب توكن)
```

### الأماكن
```
GET  /api/places?category=hotel&city=القاهرة&q=نيل&sort=top
GET  /api/places/:id
POST /api/places            (مورد فقط) { category, city, name, description, price, icon }
PUT  /api/places/:id        (المورد المالك فقط)
DELETE /api/places/:id      (المورد المالك فقط)
```

### التقييمات
```
GET  /api/places/:id/reviews
POST /api/places/:id/reviews   (يتطلب تسجيل دخول) { rating: 1-5, title, body }
```
ملاحظة: كل مستخدم يقدر يقيّم نفس المكان مرة واحدة بس.

### لوحة المورد
```
GET /api/vendor/places   (المورد فقط - أماكنه هو بس)
GET /api/vendor/stats    (مشاهدات حقيقية + متوسط تقييم + عدد التقييمات)
PUT /api/vendor/plan     { plan: "basic" | "pro" | "premium" }
```

## 5. اللي لسه ناقص (خارج نطاق هذا الباك-إند)

- الدفع الإلكتروني ونظام الحجوزات الفعلي
- رفع صور حقيقية (يحتاج تخزين ملفات مثل S3/Cloudinary)
- إرسال إيميلات (تأكيد حساب، استعادة كلمة مرور)
- لوحة تحكم إدارية (Admin) للموقع بالكامل
- ربط الفرونت إند الحالي بهذا الـ API فعليًا (الفرونت إند دلوقتي لسه بيستخدم بيانات وهمية محليًا)
